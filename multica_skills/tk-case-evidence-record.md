# tk-case-evidence-record

Use this skill when creating, auditing, or using evidence records for TikTok/TK videos, trends, account experiments, competitor posts, failed tests, and conversion examples.

## Required case fields

- Case ID
- Source type
- Source URL or local file
- Date collected
- Region/language
- Account/niche
- Content category
- Audience motive
- Viewer tension or curiosity gap
- Hook type
- First 3 seconds
- Script structure
- Filming method
- Shot sequence
- Editing rhythm
- Music/audio role
- On-screen text/caption style
- Heat signals
- Discussion signals
- CTA and conversion path
- Metrics observed
- Repeatable pattern
- Risks and policy notes
- Linked knowledge claims
- Confidence level
- Next test idea

## Case quality rules

- Separate observed facts from interpretation.
- Record missing metrics as unknown instead of guessing.
- Keep source date and collection date visible.
- Treat screenshots, links, and local files as evidence, not proof of a general rule.
- A case can support, weaken, or contradict a claim.
- Failed tests and low-performing posts are valuable counterexamples.

## Output protocol

When done, output:

- completed case record or case audit
- classification notes
- candidate knowledge claims
- counterexamples or caveats
- recommended next test
- short-term-memory.md
- long-term-memory.md
