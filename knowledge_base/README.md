# TK video knowledge base

This folder is the local long-term memory surface for the tk_video project.

The goal is to accumulate reusable content knowledge for a general TikTok/TK traffic account that may later evolve toward ecommerce, affiliate, or product-led conversion.

## Memory surfaces

- `taxonomy/TAXONOMY_V0.md`: content categories, audience motives, hooks, scripts, filming methods, edit rhythms, audio roles, trend signals, discussion signals, and conversion paths.
- `case_library/CASE_TEMPLATE.md`: standard evidence record for viral references, competitor posts, own experiments, failed tests, and conversion examples.
- `relations/CLAIM_TEMPLATE.md`: template for durable knowledge claims.
- `trends/TREND_LEDGER_TEMPLATE.md`: template for current trend observations and decay tracking.
- `predictions/PREDICTION_LEDGER_TEMPLATE.md`: template for blind hotness prediction tests.
- `evaluation/JUDGE_SCORECARD_TEMPLATE.md`: template for judge evaluation, failure attribution, and anti-overfit notes.
- `evaluation/PREDICTION_EVALUATION_PROTOCOL.md`: outcome hiding, unblinding, scoring, baseline, and failure-attribution rules for TK prediction tests.

## Agent ownership

- TK Knowledge Classifier owns taxonomy boundaries and relation applicability.
- TK Case Librarian owns case records and evidence completeness.
- TK Knowledge Modifier owns approved memory updates, backlinks, indices, confidence levels, stale-claim deprecation, and change history.
- TK Content Explainer owns knowledge-based explanations and pre-outcome hotness predictions.
- TK Viral Judge owns prediction scoring, workflow diagnosis, and anti-overfit checks.

## Rule

No durable claim should be treated as true unless it links to case evidence, has a scope, and has a confidence level.
